# PLAY: spiral-turn — one measured turn of the intelligence spiral

*Provenance: turns 22-37 (2026-07-04), 16 consecutive GAIN verdicts, two
self-caught errata classes. Every rule below was paid for, not designed.
The skill (~/.claude/skills/spiral) is the contract; this play is the
hardened execution shape.*

## The shape (5 acts, ~4-8 tool calls per turn)

1. **EVIDENCE** — the bottleneck must be something the estate demonstrably
   re-paid, lost, or risked. Sources in cost order: this session's own
   failures (best — you already paid the reproduction) > spine/ledger tails
   > open items in prior turn entries ("open": fields) > THINKING ledgers >
   journal standing items. A bottleneck you can't name a receipt for is a
   vibe — skip it.
2. **BUILD smallest** — templates/plays first; new verbs join help + dispatch
   + CAPSULE (drift-enforced); new ledgers join witness + backup AT BIRTH;
   new checks print a success token that is NOT a substring of any failure
   word. Joints (PS quoting, BOM, CRLF, git renormalization) budget double.
3. **VERIFY in a separate call** — run the artifact on its real path, read
   the output, THEN seal. The premature-seal errata: a GAIN written in the
   same call as its verify was FALSE when written. If live verification is
   possible (a real stale fact, a real race), prefer it over synthetic.
4. **SEAL through `station seal E:/station/spiral.jsonl`** — never a direct
   append (clock-stamps 't', adds by=; hand-typed times drifted +48min into
   a witnessed ledger). Entry carries: target, built, measure
   (BEFORE -> AFTER with numbers), verdict, kill (the falsifier that
   retires the artifact). NULL/REGRESS entries are as mandatory as GAIN.
5. **COMMIT + `station backup`** — one commit per turn, message = turn N +
   one line.

## Anti-patterns (each one has an errata or correction entry)

- Sealing in the same call as verifying (premature-seal).
- Typing timestamps from memory (invented-timestamp).
- Diagnosing from unfiltered tails (partial-read-claim x3; use
  spine N <match> / tally / log — the instrument beats your read).
- Accusation/root-cause claims without one refute pass (the 24601 rule;
  plays/refute.md).
- "Fixing" without measuring the broken side first — the turn-33 stress
  measured 'a'-mode BEFORE the lock existed; that number IS the measure.
- Touching Daniel's repos: read-only always fine; protecting bytes
  (rescue) fine; commits/pushes to his repos need his go-ahead.

## Multi-turn commissions (^N)

Alternate cheap/deep; keep a beat-sized item (a verification that needs
wall-clock) parked for when the clock catches up — nothing waits. At the
budget floor the correct final turn is always `station molt` + journal
narrative; ending well is a turn (the spiral climbs through molts).
