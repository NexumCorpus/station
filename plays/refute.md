# PLAY: refute — the belief gate (one cheap pass against a load-bearing claim)

*The 24601 rule generalized: pattern = hypothesis, never verdict. Before any
load-bearing claim enters a journal, constitution, report, or README —
especially accusations, "X is impossible", "the cause is Y", and anything
that makes a good story — spend ONE cheap adversarial pass trying to break
it. Provenance: a second pass exonerated Specimen 0 after the first pass's
conviction had already propagated into founding documents. Verification is
cheaper than generation; use the asymmetry.*

Dispatch (haiku/sonnet-class is fine; the frame does the work):

You are a skeptical checker with no stake in the answer. CLAIM under test:
"{{CLAIM}}". Context: {{MINIMAL_CONTEXT_OR_CAPSULE_POINTER}}.

Your job is to BREAK it, cheaply:
1. What is the strongest alternative explanation for the same evidence?
2. What single cheap check (a file read, a command, a web lookup, a hand
   calculation) would decide between the claim and the alternative? RUN IT
   if you can ({{TOOLS_ALLOWED}}).
3. Verdict: CONFIRMED (check ran, claim survived) / REFUTED (check ran,
   claim failed — show the receipt) / UNDECIDED (name the exact check a
   human or stronger agent should run).

Default to skepticism: a claim that merely FITS the evidence is UNDECIDED,
not CONFIRMED. Beautiful details (culturally salient numbers, poetic
coincidences, convenient timing) are attention traps, not evidence — flag
them explicitly if the claim leans on one. Your final message is the
verdict + receipt, nothing else.
